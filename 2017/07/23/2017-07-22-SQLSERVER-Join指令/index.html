2017.07.22 [SQLSERVER] Join指令

Posted on 2017-07-23

記得是7月初吧，前輩教我怎麼幫欄位轉換成中文，就select需要的欄位，在欄位後加上as中文名稱，例如：有個欄位叫NAME，轉換成中文就是：

select NAME as’名稱’ from table

AS語法很簡單，所以今天要講得是Join。

一開始我一直不懂Join為什麼分成3種：LEFT (OUTER) JOIN、INNER JOIN、RIGHT (OUTER) JOIN，有什麼不一樣的效果。ex:LEFT JOIN是撈到資料表放左邊，或者撈左邊資料。而前輩有我解釋LEFT JOIN，因為他說工作這麼久，他用到最多就是LEFT JOIN，其次是INNER JOIN。RIGHT JOIN和LEFT JOIN概念一樣，只是一個以左邊表為主，另一個以右邊表為主。但前輩跟我解釋了很久我只記得前輩說LEFT JOIN是以左邊的表為主，我一直想說左邊的表是指？後來實作之後才發現原來那麼簡單，也覺得請教問題之前要先理解問題，真的沒辦法理解再問。

我以動物園的地區資料表、部門資料表做範例。

Zone地區

select *FROM Zone
![Alt text](/2017/07/23/2017-07-22-SQLSERVER-Join指令/tb_zone.png) Department部門 
select *FROM Department
![Alt text](/2017/07/23/2017-07-22-SQLSERVER-Join指令/tb_department.png)
LEFT JOIN
SELECT Zone.ZONE_AREA,Department.DEP_NAME from Zone
left join Department on Department.DEP_CODE = zone.DEP_CODE

INNER JOIN

SELECT Zone.ZONE_AREA,Department.DEP_NAME from Zone
inner join Department on Department.DEP_CODE = zone.DEP_CODE
